üéµ ‚Äî Song Notice ‚Äî üéµ
Part of the song loader rewrite involves faster song analyzing and caching. GHWTDE 1.1.1.0 fundamentally changes the way that hashes are calculated for songs!

‚ùì Why does this matter? ‚ùì
- In order to make sure that online setlists and songs are in sync, each song has a unique checksum value based on the contents of its .pak file.
- Songs in 1.1.1.0 and later will have DIFFERENT CHECKSUMS than previous versions!
- You will NOT be able to play custom songs online with members on older versions!

üìÅ Will this change how mods are handled? üìÅ
NO, this will not change the way that YOU as a user interact with mods! Handle your mods the same way that you have been!
- Mods will be installed the same way, by placing them in DATA/MODS
- Song checksums for AutoPlay and charters will remain the same, and tutorials should stay the same (for now)

üéâ Update your mod, and enjoy! üéâ

May 25, 2023 - Mod Loader Rewrite PSA

Author: Zedek the Plague Doctor ‚Ñ¢
The mod loader is in the process of being completely rewritten from scratch! This will take some time (not a ton) but will provide benefits like:

Much faster mod loading
- Mods should (hopefully) load a lot faster!
- GHWTDE.dll and the game will now speak back and forth as little as possible
- Logic changes in the update loop to hopefully make the process smoother

ScriptMod improvements
- .qb files in ScriptMods can now be loaded recursively!
- DATA/MODS/MyMod/LevelA/LevelB/CodeFile.qb.xen works!
- .qb files no longer need to be named the same as the folder they are in
- _Load functions in the .qb file will be prefixed with the name of the .qb file (see above)

Cleaner Backend
- Cleaner DLL source code allows for easy addition of future mod types

Mod Limit
- Mod loading uses dynamic values and is no longer restricted to a static list
- Theoretically infinite mod count (no longer restricted to 30,000 or so)

Stay tuned!
